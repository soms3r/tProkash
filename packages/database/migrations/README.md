# Migrations

This directory is auto-generated by **drizzle-kit**.

## Structure

```
migrations/
  meta/
    _journal.json     — Tracks migration order and applied state
    _schemas/         — Schema snapshots for each migration
  0000_<name>/       — Migration folder (e.g., 0000_initial)
    meta.json         — Drizzle metadata
    snapshot.json     — Schema snapshot
    down.sql          — Rollback SQL (drizzle-kit generates this)
```

## Workflow

1. Define/update tables in `src/schema/` using Drizzle ORM.
2. Run `pnpm --filter @tprokash/database db:generate` to produce a migration.
3. Review the generated SQL in the migration folder.
4. Run `pnpm --filter @tprokash/database db:migrate` to apply.

## Commands

| Command | Description |
|---------|-------------|
| `db:generate` | Generate a migration from schema changes |
| `db:migrate`  | Apply pending migrations |
| `db:check`    | Check for pending schema changes (dry-run) |
| `db:push`     | Push schema directly (dev only — no migration files) |
| `db:studio`   | Open Drizzle Studio |

## Notes

- Do NOT manually edit files in `meta/` — drizzle-kit owns them.
- Migration folders are named by drizzle-kit with sequential indices (0000, 0001, ...).
- Always review generated SQL before applying to production.
